FAQ

Frequently asked questions

Short answers. For the full guide, see the docs.

What is Glimmer?

Glimmer is an AI-first image editor that runs in your browser. It’s a genuine layer-based editor built on a custom WebGL2 engine — brushes, masks, adjustment layers, the works — with a generative AI suite (generate, edit, cutout, outpaint, relight and more) and a chat assistant woven directly into the canvas.

Is it free? Is it open source?

The source is public on GitHub and built to be self-hosted. You run your own stack and bring your own AI provider key, so your only cost is the provider usage you choose to spend.

Do I need an API key, and what does it cost?

The AI features call models through OpenRouter, so you provide an OpenRouter key in your server’s .env. You’re billed by OpenRouter for what you use; Glimmer keeps a per-job credit ledger so you can see and cap spend. The non-AI editor — layers, brushes, adjustments, crop, export — needs no key at all.

Which AI models does it use?

By default, Google’s Gemini image model via OpenRouter for generation & editing, and a text model for the assistant’s planning. Background cutout and “magic select” run in your browser with on-device ML when possible (with a server fallback). Models are configured by environment variable, so you can swap them.

Where do my images go? Is it private?

You host everything. The editing engine runs entirely in your browser. AI actions upload only the needed region to your storage (MinIO) and call the provider from your server — provider keys never reach the browser. Local-only operations (cutout, magic-select) don’t leave your machine.

Can I use it as a normal editor, without AI?

Absolutely. Open or paste an image and use the full toolset — selections, layers & masks, brushes, clone/heal, dodge/burn, adjustment layers, filters, crop, transform, type, shapes — none of it requires an AI key.

Is this a real editor or just a wrapper around a model?

A real editor. The core is a hand-written WebGL2 compositor with linear-light blending, tiled documents, the full blend-mode set, GPU brushes, layer masks and smart filters. AI is a first-class tool alongside everything else — each AI result is a non-destructive layer you can mask, blend and undo.

What’s the difference between the “omni” and “classic” layouts?

Omni (the default) is canvas-first: a full-screen canvas with one omnibar and minimal floating chrome, ideal for laptops. Classic is the familiar docked layout with a menu bar and side panels. They share the same document — switch any time. Everything in the classic menus is also reachable via the ⌘K palette.

What browsers does it support?

A modern desktop browser with WebGL2 — current Chrome, Edge, Firefox or Safari. The on-device AI (cutout / magic-select) prefers WebGPU and falls back to WebAssembly. Glimmer is designed for a desktop/laptop workflow.

Does it work on mobile or tablet?

It runs, but Glimmer is built for a pointer + keyboard workflow on a larger screen. Phones aren’t a target right now.

How do exports and project files work?

Flatten and download as PNG, JPEG or WebP (with a quality choice via “Export as…”). To keep your layers, save the whole document as an .aips project and re-open it later.

How do I run my own instance?

Clone the repo, copy .env.example to .env and add your OPENROUTER_API_KEY, then docker compose up for the API, workers and storage, and run the web client with pnpm. The full walkthrough is in the self-hosting docs.

What’s it built with?

A pnpm monorepo: React + TypeScript + a custom WebGL2 engine on the front end, a Fastify API, Python Celery workers for the heavy AI/image jobs, and Postgres + Redis + MinIO behind Docker Compose. Routing to providers (OpenRouter, with optional specialists) goes through one capability interface.
